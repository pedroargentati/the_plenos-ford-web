Revolução no Abastecimento

Imagine nunca mais se preocupar em abastecer seu veículo com combustível adulterado. Com o Ford Fuel, essa preocupação se torna coisa do passado. Nosso projeto traz uma proposta revolucionária para a indústria de combustíveis brasileira, com o compromisso de tornar a experiência de abastecimento mais segura, transparente e confiável para todos.

No Brasil, a comercialização de combustível adulterado é um problema persistente que afeta milhões de motoristas, o meio ambiente e a economia. A adulteração de combustíveis compromete o desempenho dos motores, levanta preocupações sobre a segurança nas estradas e contribui para a poluição do ar. É um problema que exige uma solução inovadora, e o Ford Fuel está aqui para oferecer exatamente isso.

Nossa abordagem única baseia-se em três pilares essenciais:

Histórico Abastecimento

  • Ford Pass: Implementação da funcionalidade ao aplicativo.
  • Localização: Informações da localização do abastecimento.
  • Informações: Informações sobre o combustível do abastecimento.
  • Disponibilidade: Permanencia do histórico de abastecimento ao app.

Denúncia Automática

  • Periodização: Relatórios de denúncia gerados mensalmente.
  • Indicadores: Indicadores que permitem a ampliação de ciencia.
  • Relatórios: Dashboards informativas com indicadores gerais.
  • Automatização: Automatização da atualização de dados .

Parcerias com Postos

  • Contatação: identificação e abordagem desses estabelecimentos.
  • Parceria: Usuário autenticado via token de banco de dados
  • Reconhecimento: Pesquisas no banco para obtenção desses.
  • Divulgação: Maximização do alcance e o impacto da iniciativa. e diferenciais.

Arquitetura de Solução

1

Bem-vindo à era revolucionária do abastecimento de combustível com a arquitetura Ford Fuel. Estamos propondo uma transformanção geral na maneira é exergado o abastecimento, combinando tecnologia de ponta com parcerias estratégicas para garantir o fim da comercialização do combustível adulterado e a prevalencia unica e exclusiva de combustível de qualidade. Neste guia, vamos explorar em detalhes como nossa arquitetura pode vir a moldar futuro, tornando o abastecimento mais transparente e confiável

Obtenção de Dados

A Ford, com seus investimentos em tecnologia, apresenta inúmeros sensores equipados em seus veículos e dois deles desempenham um papel crucial em nossa iniciativa, o sensor de coordenadas e o sensor de qualidade de combustível. Estes são os dados que iremos necessitar em nossa solução...

Envio de Dados

A TMC (Transportation Mobility Cloud) é a tecnologia interna da Ford que desempenha o papel de transferência de dados provenientes desses sensores para o banco de dados...

Armazenamento de Dados

Dado que não temos acesso à modelagem do banco de dados da Ford, criamos um banco de dados de demonstração usando o Big Query, uma tecnologia já adotada pela Ford. Sua modelagem e script de criação estão disponíveis para download...

Obtendo o Endereço

Para enriquecer o histórico de abastecimento, construímos uma API em Java onde, ao enviarmos as coordenadas ao serviço OpenCage o mesmo nos retorna informações do endereço como Rua, Bairro, Cidade, Estado e CEP. Esses dados, ao serem retornados são armazenados no banco de dados através de outra API, tambem em JAVA. (Saiba mais...)

Nome do Posto

Com os dados do endereço em mãos, podemos agora obter o nome do posto. Com as coordenadas aliadas aos ultimos dados obtidos, nossa API em Python utiliza uma lógica de web-scraping para encontrar o nome do estabelecimento. Como nenhuma coordenada retorna o número do endereço, nosso retorno está sucetível à falhas, e é aí que entra o próximo passo...

Confirmação do Nome

Dentro da mesma requisição, montamos uma lógica de machine learning que, aliada ao web scrapping tende a aprender e aprimorar as respostas dadas, eliminando os erros ao longo do tempo. Ao final, esses dados são retornados e armazenados no banco com uma outra API em Java...

Disponibilização dos Dados

Com esses dados armazenados, agora podemos apresentar ao clientes pela correta segmentação de veículo. Para tal, desenvolvemos uma réplica das telas do aplicativo Ford Pass com suas devida modificações para a implementação do Ford Fuel. Para tal, utilizamos Dart, Flutter e construimos o back-end de comunicação com SpringBoot em Java... Nele, o usuário podera ver os dados citados anteriormente...

Denúncia de Adulteração

Com os registros de abastecimetos armazenado no Banco, criamos uma conexão entre o Big Query e o Power BI. Neste construimos uma dashboard com indicativos relevantes que facilitarão a abordagem dos órgãos fiscais

Parceria com Postos

Com os registros de abastecimento no banco, através de consultas SQL poderemos selecionar os postos que tiverem os melhores registros contactando-os e oferecendo parcerias.

Por que utilizar Ford Fuel ?

Nossa solução propõe a integração do medidor de qualidade de combustível diretamente no sistema dos carros. Isso significa que os motoristas terão acesso imediato às informações sobre a qualidade do combustível no painel do carro, tornando o processo mais conveniente e rápido

Análise de Dados e Geração de Relatórios

Coletamos e armazenamos dados sobre a qualidade do combustível ao longo do tempo, permitindo a geração de relatórios detalhados e a análise de tendências. Essas informações podem ser utilizadas para identificar padrões, otimizar o desempenho dos veículos e tomar decisões informadas sobre a escolha de postos de combustível.

Integração com o Aplicativo FordPass

O FordPass é um aplicativo inovador que oferece uma variedade de recursos para os motoristas, com a integração da nossa solução, os usuários poderão acessar facilmente informações cruciais sobre a qualidade do combustível diretamente em seus smartphones.
